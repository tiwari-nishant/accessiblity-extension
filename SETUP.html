📦 Prerequisites

Before you begin, ensure you have the following installed:

Required Software

  • Node.js (v14 or higher) - Download here
  • npm (comes with Node.js)
  • Firefox Browser - Download here
  • Git (optional) - Download here

Verify Installation

Open a terminal and run:

node --version
npm --version

You should see version numbers for both commands.

🔧 Part 1: Backend Setup

1Create Project Directory

# Create a new directory for the project
mkdir accessibility-scanner
cd accessibility-scanner

2Initialize Node.js Project

# Initialize package.json
npm init -y

3Install Dependencies

# Install required packages
npm install accessibility-checker puppeteer ws exceljs
📦 Packages Installed:
  • accessibility-checker - IBM's accessibility testing tool
  • puppeteer - Headless browser automation
  • ws - WebSocket server for browser extension
  • exceljs - Excel report generation

4Create Configuration Files

Create .achecker.yml

Create a file named .achecker.yml in the project root:

# Accessibility Checker Configuration
policies:
  - IBM_Accessibility

reportLevels:
  - violation
  - recommendation
  - manual

outputFormat:
  - json
  - html
  - csv
  - xlsx

outputFolder: results
outputFilenameTimestamp: false

label:
  - OpenBMC-Web-UI

Create .gitignore

node_modules/
results/
*.log
.DS_Store

5Create Backend Scripts

You need to create three main scripts:

  • scan.js - Main CLI scanning script
  • websocket-server.js - WebSocket server for browser extension
  • generate-report.js - Excel report generation utility
⚠️ Note: Download these files from the project repository or copy them from the existing setup.

6Update package.json Scripts

Edit your package.json and add these scripts:

{
  "scripts": {
    "scan": "node scan.js",
    "server": "node websocket-server.js",
    "report": "node generate-report.js"
  }
}

7Test Backend Installation

# Test the WebSocket server
npm run server
✅ Expected Output: 
WebSocket server running on ws://localhost:8080
Waiting for browser extension to connect...

Keep this terminal open. Open a new terminal for the next steps.

🔌 Part 2: Browser Extension Setup

1Create Extension Directory Structure

mkdir -p browser-extension/sidebar
mkdir -p browser-extension/icons

2Create Extension Files

Create manifest.json

Create browser-extension/manifest.json:

{
  "manifest_version": 2,
  "name": "Accessibility Scanner",
  "version": "1.0.0",
  "description": "Automated accessibility scanning",
  
  "permissions": [
    "activeTab",
    "tabs",
    "downloads",
    ""
  ],
  
  "background": {
    "scripts": ["background.js"]
  },
  
  "sidebar_action": {
    "default_title": "Accessibility Scanner",
    "default_panel": "sidebar/sidebar.html"
  },
  
  "browser_action": {
    "default_title": "Open Accessibility Scanner"
  }
}

Create background.js

Create browser-extension/background.js:

// Open sidebar when browser action is clicked
browser.browserAction.onClicked.addListener(() => {
  browser.sidebarAction.open();
});

Create Sidebar Files

Create the following files in browser-extension/sidebar/:

  • sidebar.html - Sidebar HTML structure
  • sidebar.css - Sidebar styles
  • sidebar.js - Sidebar logic and WebSocket client
⚠️ Note: Download these files from the project repository or copy them from the existing setup.

3Create Extension Icons

Create icon files in browser-extension/icons/:

  • icon-16.png (16x16 pixels)
  • icon-32.png (32x32 pixels)
  • icon-48.png (48x48 pixels)
  • icon-128.png (128x128 pixels)

Quick Icon Creation

For testing, you can:

  1. Use any image editor to create simple colored squares
  2. Add text "A11y" or an accessibility symbol
  3. Export in the required sizes
  4. Or use placeholder icons from placeholder.com

4Load Extension in Firefox

  1. Open Firefox
  2. Navigate to about:debugging#/runtime/this-firefox
  3. Click "Load Temporary Add-on..."
  4. Navigate to your browser-extension folder
  5. Select the manifest.json file
  6. Click "Open"
✅ Success! The extension should now appear in your Firefox extensions list.

5Open the Extension Sidebar

  1. Click the extension icon in the Firefox toolbar
  2. The sidebar should open on the left side
  3. You should see "Accessibility Scanner" interface

🚀 Part 3: Usage

Using the Browser Extension

Step-by-Step Process

  1. Start the WebSocket Server 
    npm run server
    Keep this running in a terminal.
  2. Open Your Web Application
    • Navigate to your localhost application (e.g., http://localhost:8000)
    • URL must contain "localhost" or "stglabs"
  3. Open the Extension Sidebar
    • Click the extension icon or press Ctrl+B / Cmd+B
    • Sidebar should show "Connected" status
  4. Run a Scan
    • Click the "Run Scan" button
    • Watch progress as it scans all routes
    • View real-time logs and statistics
  5. View Reports
    • Check the results/ directory
    • Reports available in HTML, JSON, CSV, and XLSX formats

Using the CLI Scanner

For standalone scanning without the browser extension:

npm run scan

This will scan all configured routes and generate reports in the results/ directory.

Generating Excel Reports

To create a consolidated Excel report:

npm run report

🔧 Troubleshooting

Backend Issues

❌ "Cannot find module 'accessibility-checker'"

Solution:

npm install

❌ "Port 8080 already in use"

Solution on macOS/Linux:

lsof -ti:8080 | xargs kill -9

Solution on Windows:

netstat -ano | findstr :8080
taskkill /PID  /F

❌ WebSocket server not connecting

Solutions:

  • Ensure the server is running (npm run server)
  • Check that no firewall is blocking port 8080
  • Verify server shows "WebSocket server running on ws://localhost:8080"

Browser Extension Issues

❌ Extension not loading

Solutions:

  • Verify all files are in correct directories
  • Check browser console for errors (F12 → Console)
  • Ensure manifest.json is valid JSON

❌ "Disconnected from server"

Solutions:

  • Start the WebSocket server: npm run server
  • Refresh the extension sidebar
  • Check browser console for connection errors

❌ "Invalid URL" error

Solutions:

  • Ensure you're on a page with "localhost" or "stglabs" in the URL
  • Extension only works with these domains for security

Scan Issues

❌ Scans timing out

Solutions:

  • Increase timeout in websocket-server.js
  • Check if target website is accessible
  • Verify authentication credentials if required

❌ No reports generated

Solutions:

  • Check the results/ directory exists
  • Verify .achecker.yml configuration
  • Look for errors in server terminal

⚙️ Configuration

Customizing Routes to Scan

Edit websocket-server.js and modify the ROUTES_TO_SCAN array:

const ROUTES_TO_SCAN = [
    '/your-route-1',
    '/your-route-2',
    '/your-route-3',
    // Add more routes...
];

Customizing Authentication

Edit websocket-server.js and update the login configuration:

const LOGIN_CONFIG = {
    url: 'http://localhost:8000/#/login',
    usernameSelector: '#username',
    passwordSelector: '#password',
    submitSelector: 'button[type="submit"]',
    username: 'your-username',
    password: 'your-password'
};

Customizing Accessibility Rules

Edit .achecker.yml to change:

  • Policies: IBM_Accessibility, WCAG_2_1, etc.
  • Report levels: violation, recommendation, manual, potential
  • Output formats: json, html, csv, xlsx
